Getting Started

Everything you need to know to start optimizing your Raspberry Pi's boot time.

Prerequisites

  • A Raspberry Pi (3, 4, or 5) running Raspberry Pi OS (or any Debian-based Linux)
  • SSH or terminal access with sudo privileges
  • Basic familiarity with the Linux command line
  • Git installed (optional, for cloning the repository)

Quick Start

1

Clone the repository

Download the optimization script to your Raspberry Pi.

bash
git clone https://github.com/ajayduraisamy/Raspberry-Pi-Boot-Time-Optimization.git
cd Raspberry-Pi-Boot-Time-Optimization
2

Measure baseline boot time

Run systemd-analyze to record your current boot time before making any changes.

bash
systemd-analyze
3

Run the optimization script

Execute the script with sudo to automatically disable all unnecessary services.

bash
chmod +x optimization.sh
sudo ./optimization.sh
4

Reboot and verify

Restart your Raspberry Pi and check the new boot time.

bash
sudo reboot
# After reboot:
systemd-analyze
💡 The entire process takes less than 5 minutes. You do not need to edit any configuration files manually.

Architecture

System architecture and boot process overview.

Boot Process Overview

The Raspberry Pi boot process follows a structured pipeline from power-on to the login prompt:

🔌 Power On — Voltage regulator stabilizes
↓
💾 Bootloader (EEPROM) — Loads firmware
↓
🐧 Kernel Boot — Decompresses and initializes
↓
⚙️ systemd Init — Starts all services ← Our focus
↓
🖥️ Display Manager — Login screen

Service Dependencies

systemd manages services as units with dependency chains. Disabling a service may affect others depending on it. The services we target have no critical dependents in a standard desktop setup.

📐 Optimization Principle

We only disable services that are non-essential for basic desktop operation. All core system functionality (networking, audio, file systems, security) remains untouched.

How It Works

Deep dive into the optimization process, analysis techniques, and expected outcomes.

The Boot Analysis Process

systemd provides built-in profiling tools that reveal exactly where time is spent during startup:

systemd-analyze

Shows total boot time broken down into kernel and userspace phases:

pi@raspberrypi ~ $ systemd-analyze
$ systemd-analyze
Startup finished in 15.331s (kernel) + 8.442s (userspace) = 23.773s

systemd-analyze blame

Lists all services sorted by their startup time, from slowest to fastest:

pi@raspberrypi ~ $ systemd-analyze blame
$ systemd-analyze blame | head -10
4.2s NetworkManager-wait-online.service
3.1s cups.service
2.4s avahi-daemon.service
1.8s bluetooth.service
1.2s wayvnc.service
0.8s ssh.service
0.6s systemd-logind.service

Why These Services?

Each service chosen for optimization was evaluated against three criteria:

⏱️

High Boot Impact

Service adds 1+ second to boot time — significant relative to a 15-second total.

🔌

Non-Essential

Disabling it does not break core system functionality like networking, SSH, or the desktop.

🔄

Easily Reversible

Services can be re-enabled with a single systemctl enable command at any time.

What Happens When You Disable a Service?

systemctl disable removes the symlinks that cause a service to start automatically at boot. The service remains installed and can be started manually or re-enabled later. No files are deleted.

Optimization Script

Complete documentation of the optimization.sh script — how it works and what each part does.

Script Overview

The script automates the entire boot optimization process. It is designed to be safe, idempotent, and transparent — showing exactly what it does at every step.

Full Script

bash 
#!/bin/bash
# Raspberry Pi Boot Optimization Script

echo "=================================="
echo " Raspberry Pi Boot Optimization "
echo "=================================="

echo ""
echo "Boot time before optimization:"
systemd-analyze

echo ""
echo "Disabling unnecessary services..."

sudo systemctl disable NetworkManager-wait-online.service
sudo systemctl disable cups
sudo systemctl disable avahi-daemon
sudo systemctl disable bluetooth.service
sudo systemctl disable wayvnc.service

echo ""
echo "Services disabled successfully."

echo ""
echo "Current status:"

systemctl is-enabled NetworkManager-wait-online.service
systemctl is-enabled cups
systemctl is-enabled avahi-daemon
systemctl is-enabled bluetooth.service
systemctl is-enabled wayvnc.service

echo ""
echo "Optimization completed."
echo "Please reboot the Raspberry Pi and check:"
echo "systemd-analyze"

Script Breakdown

1

Banner & Baseline

Prints a header and runs systemd-analyze to capture the current boot time as a baseline for comparison.

2

Service Disabling

Executes sudo systemctl disable on all 5 target services. Each disable command removes the boot-time symlink.

3

Verification

Runs systemctl is-enabled on each service to confirm the disable operation succeeded. All should return disabled.

4

Completion

Prints a summary and instructs the user to reboot and re-run systemd-analyze to see the improvement.

Troubleshooting

Solutions to common issues encountered during the optimization process.

Permission Denied

bash
$ systemctl disable cups
Failed to disable unit: Access denied

Solution: All service disabling commands require sudo.

bash
sudo systemctl disable cups

Service Not Found

bash
Failed to disable unit: Unit file cups.service does not exist.

Solution: The service name may differ on your distribution. List all services to find the exact name:

bash
systemctl list-units --type=service | grep -i cups

No Improvement After Reboot

Solution: Verify services are actually disabled:

bash
systemctl is-enabled NetworkManager-wait-online.service
systemctl is-enabled cups
systemctl is-enabled avahi-daemon
systemctl is-enabled bluetooth.service
systemctl is-enabled wayvnc.service

If any return enabled, re-run the script with sudo.
